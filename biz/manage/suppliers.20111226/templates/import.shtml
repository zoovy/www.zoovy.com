Directions:
  • Look in the file/url your supplier automatically generates
  • If the file/url doesn't contain the correct Zoovy field names (as described below), map the columns in the "Data Position Definitions" section so Zoovy can automatically pick them up.
  • Leave columns blank that you do not want imported
  • Only lines which contain a valid SKU will be imported (all other lines will be ignored)
  • If your file/url contains a valid header with Zoovy fields, there is no need to map your Data Positions.
  • Products are currently importly manually, so when your Supplier makes a change to their products, you will need to re-import their data.


    Header Names:
  • %PRODUCTID
  • %INVENTORY
  • %CATEGORY
  • zoovy:prod_name - the products short name or title, should be less than 100 characters
  • zoovy:prod_desc - the products description, should be less than 1024 characters.
  • zoovy:prod_image1 - a URL or Image Library name which represents the primary image for the product, also use zoovy:prod_image2 .. etc.
  • zoovy:prod_image2 - a URL or Image Library name which represents the secondary image
  • zoovy:prod_thumb - a URL or Image Library name which represents the thumbnails
  • zoovy:base_price - the price, in US dollars without the dollar sign.
  • zoovy:base_weight - the weight in ounces.
