Site Integrator Configuration

(Only available on themes with an navigation sidebar on the left -->
Website URL:
This URL is provided throughout your site as your website address (but NOT used for rewritten urls below). Typically this is used for channels, etc. It is provided here simply as a reference.

> Enable Zoovy Site Integrator Features

> Break out of frames by adding target="_top" to Checkout button
Use this if you encounter SSL errors while using a custom theme.

> Enable Soft Cart functionality
Allows html pages on other sites to pass price, description, and taxable information to the cart using buy me buttons. WARNING: enabling this feature significantly reduces the security of your Zoovy store. You will need to configure referrer URLS as well

> Enable Product Redirection.
Product Redirection lets you map http://yourname.zoovy.com/product/PRODUCTID to a different URL in setup (Typically enabled for Site Integrator accounts, and Disabled for Hybrid accounts).

> Rewrite all Zoovy URLS to SSL server.
Enable this feature if you are using Zoovy Site Integrator inside a secure frameset - warning this will make all pages much slower, and breaks most domain redirections. Please only enable this under instruction of a Zoovy representative.

> Disable Website Navigation Root Categories
By default Zoovy places category links on each page wrapper to the top level categories in your website, if you have no categories, or plan to create your own category buttons then disable this feature.

> Disable Website Navigation Sub-Categories
Sub categories appear within the root categories, disable this if you plan to create your own hyperlinks to categories.
NOTE: this setting only affects "SUBCAT" elements, if you have a SUBCAT in your wrapper - those will still display. (If you don't want this behavior, then pull the SUBCATS out of your wrapper)

> Disable "Continue" button in shopping cart, and Cancel button on products.
By default the continue button shopping button appears on the cart and links to the homepage, or a URL you specify below. Disabling this will also disable the continue shopping (commonly referred to as the "cancel button") which is displayed on a product page.

> Disable Cart Link back to Products
By default the Zoovy cart will provide a link to the products placed in the cart. (Normally enabled for Site Integrator customers, disabled for Hybrid accounts.)

> Look for meta information passed on non-cart pages (slower).
If you link directly to the Zoovy site using META (affiliate tracking) then enable this feature, note: this does not apply to those using redir.cgis.

> Disable HTML stripping from product descriptions.
By default Zoovy will remove any HTML from product descriptions before placing them into product lists.
> EXPERIMENTAL: Allow dashes in product id's.
By default Zoovy uses dashes in the SKU to delimit where the product id ends and the product options begin. Dashes in the product id are translated to underscores (_). For merchants who DO NOT USE OPTIONS this feature allows dashes to be placed in the product id. This feature is currently experimental, and may result in some features in Zoovy not behaving properly. You should notify support before you enable this feature.
> Allow negative amounts in the shopping cart.
Handy if you plan to use Incomplete Sales as gift certificates.
Zoovy Logo:
Claim Flow: Leave this blank to use the product flow (or a blank page, if no default flow is selected)
> No claim message on claim page.
The claim message is set in site / automated settings and by default is displayed before the body of the page starts. If this interferes with your custom flow, then disable this setting.
Home Button: Default: /
Enter the full URL path where the home button should go, only necessary if you are using a Zoovy standard theme.

Continue Shopping: Default: /
The continue shopping button appears as a standard button (which can be disabled above) on the shopping cart page, enter the full path you wish to send the customer to when it is clicked. (Usually this will be your homepage)

Search: Default: /search.cgis
The default URL for searching is http://yourname.zoovy.com/search.cgis, if you have created your own site search then enter the full URL here.

About Us: Default: /company_info.cgis
The about us link is a standard button available on most Zoovy themes, it is only necessary to define this if you plan on using a standard Zoovy theme.

News*: Default: /news.cgis
The news link is a custom button available on some Zoovy themes, it is only necessary to define this if you are using a specialized Zoovy theme which includes the news link.

Return Policy: Default: /returns.cgis
The return policy link is a standard link available on most Zoovy layouts, it is only necessary to define this if you plan on using a standard Zoovy layouts.

Privacy Policy: Default: /privacy.cgis
The privacy policy link is a standard link available on most Zoovy layouts, it is only necessary to define this if you plan on using a standard Zoovy layouts.

Contact Us: Default: /feedback.cgis
The contact us link is a standard button available on most Zoovy layouts, it is only necessary to define this if you plan on using a standard Zoovy themes.

Logout Path: Default: /
This is the page customers are sent when they have been accessing their account at http://yourname.zoovy.com/customer_main.cgis and then decide to press the log out button. Normally this will be your homepage and will always be targeted to the _top frame to insure no accidental frame wrapping occurs.

The contents of the side bar HTML appear exclusively in standard layout 2 themes, under the root navigation categories on the left hand side of the screen.
> Display sidebar on secure pages
This option should only be enabled if the sidebar contains ONLY HTTPS graphics. If this option is enabled with HTTP graphics then the customers web browser will display an error telling the customer that the checkout contains insecure information.

Allowed Referrers:
Referrer URLS are used to verify that a browser which is adding items to the softcart is coming from an allowed page. - Use a blank line with the word ANY to allow all referrers (very insecure)
