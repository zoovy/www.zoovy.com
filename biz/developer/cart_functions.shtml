Zoovy Developer Cart Functions

 

Using the HTML code in the examples below, you can allow users browsing a site on any server on the internet to add items to a shopping cart in your Zoovy store.

 

In the HTML examples below, anywhere you see a bold number, this should be replaced with the product id for that product (this is the unique identifier for the product in the Zoovy system, which is not necessarily the same as your product’s name).  You can find a product’s id by logging into your Zoovy account and selecting the product in the product management section.  The product id will follow the word “Product: “ at the top of the page.

 

Anywhere you see “merchant_id”, exchange that for your Zoovy login.  All of the HTML examples shown are case-sensitive (capitalization matters) so make sure to use the forms exactly as show (but feel free to change anything that appears in italics to suit your tastes).

 

Product Options

 

Product options (or “pogs”) in the Zoovy system allows you to manage different versions of the same product easily.  If you have something like a tee-shirt, which has different colors and different sizes, pogs are an easy way to keep track of things.  You can create and edit product options by logging in to the Zoovy system, going to the product management section, and selecting the “Manage Options” button for the product you’d like to change.

 

IMPORTANT INFORMATION IF YOUR’E USING PRODUCT OPTIONS:

If you have product options for configured for an item you’re selling, the HTML form you choose below MUST include the product options, or it will not be added to the cart!  Our system has safeguards that prevent bogus products from being entered. 

 

You’ll need some extra information from Zoovy to do an “Add to Cart” for any product with Pogs.  To get to this information, log in to the Zoovy web site and go to the product management section and select “Buy Me Button” for the product you wish to use.

 

For any “pogs” or “pogs:0001” field, you MUST use the value from the “pogs” input tag.  For every “pog_pogname” selection you see in the “Buy Me Button” section, you MUST include it in the following forms as per the example. Be aware that you MUST have the options configured in the Zoovy system if you include the “pogs”, ”pogs:0001”, “pog_pogname” or “pog_pogname:0001” fields in the form, and if you ever change your product options in the Zoovy system you MUST change your HTML to match the new setup!

Note: You can include any valid HTML between the a href tag, including image links

Using Product Options / Notes with Multiple-Item Forms

 

Multiple add to cart forms are made to act much like the single add to cart forms, and can have all of the same things like product options and notes.  There are some changes that need to happen when you create the form for it to work properly:

 

·        The input names will be in the form of “field:0001” instead of just “field”.  For example, “pogs” turns into “pogs:0001” for product id 0001, and “pog_Size” will be “pog_Size:0001”, “notes_prompt” will be “notes_prompt:0001”, etc.

·        The “product_id” field is replaced by the “product_id:0001” field.  The value for this field should always be “1”, instead of the product’s id.

 

Submit Buttons

 

Anywhere you see an input type=”submit” (like the following):

 

<input type=”submit” value=”Add Checked Items to Cart”>

 

You can use the following syntax instead to use an image instead of a grey HTML form button:

 

<input type=”image” src=”http://your_image_url”>

 

Where “http://your_image_url” is the URL to the image you want to use.  If you know the width and height of the image being used, you can specify them there as well:

 

<input type=”image” src=”http://your_image_url” width=”200” height=”50”>

 

Also, you can include as many “submit” or “image” buttons as you want in the form tags, they will all cause the same thing to happen.  This is handy if you want to have a way to add checked items to the cart at both the top of the page and the bottom of the page, or next to every product in a multiple-product list.

 

Product Notes

 

Every “notes” field must have the “notes_prompt” field accompanying it as shown in the examples above…  this lets the Zoovy system know what to put in the description in front of the text the user entered.  You can make a large textbox instead of a single-line textbox by changing the following:

 

<input type=”text” name=”notes” value=”Default Text”>

 

Into this (adjust the “cols” and “rows” to make the box bigger or smaller):

 

<textarea cols=”50” rows=”10” name=”notes” wrap=”soft”>

Default Text

</textarea>

 

Changing Quantities

 

You can change the values of inputs with a name of “quantity” or “quantity:0001” to your liking.  For instance if you have a checkbox add to cart, changing the value= attribute part of the “quantity” input tag will change the quantity of that item that will be added to the cart when that product is checked.  If you have a form that allows the user to input a quantity, changing the value will result in the form saying something other than “0” when its displayed.

 

Meta information

 

Corporate customers who retrieve data from the Zoovy system can add information into the order that you can be checked for later on (like an identifier for who referred the customer, etc) by parsing the product through the developer API.  To add meta information into the cart, add the following inside the form tags to the HTML:

 

<input type=”hidden” name=”meta” value=”secret info”>

 

The information “secret info” will be stored as an invisible item in the shopping cart named !META that can then be retrieved later.  This feature is not recommended for non-programmers who do not want to edit or view their orders through their own programs.
